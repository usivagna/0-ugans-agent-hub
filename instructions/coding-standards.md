# Coding Standards

Reusable instruction set for enforcing coding standards across projects.

## Usage

Copy this into a repo's `.github/copilot-instructions.md` or reference when prompting Copilot.

---

## Instructions

- Follow the project's existing code style and conventions
- Write self-documenting code with clear variable and function names
- Keep functions under 30 lines where possible

- Write pure functions when possible; minimize side effects
- Use typed parameters and return types in languages that support them
